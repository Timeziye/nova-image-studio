# 基于作者原版项目的改动说明

本文档记录本仓库相对于作者原版 Nova Image Studio 项目做出的主要改动和新增功能，便于后续维护、部署和功能回溯。对比基准为 `tianjiangqiji/nova-image-studio` 的 `main` 分支。

## 界面与布局

- 解除宽屏入口的 `xl` 宽度限制，中等宽度设备和平板横屏下也可以看到并使用宽屏按钮。
- 宽屏模式下增加侧栏折叠/展开状态，折叠状态会保存在本地。
- 工作区侧栏会合并显示服务端队列与画布对应生成队列的排队状态。

## 我的素材

- 新增素材文件夹能力：支持新建文件夹、重命名文件夹、删除文件夹。
- 删除文件夹时，会连同文件夹内图片一并删除。
- 支持将图片归档到指定文件夹，也支持移回未归档。
- 支持导入整个本地文件夹，导入后自动创建或复用同名素材文件夹。
- 支持文件夹拖动排序和文件夹内图片拖动排序。
- 拖动排序会自动切换为自定义排序，不需要用户先手动切换排序模式。
- “全部”页签用于展示文件夹和未归档素材；打开具体文件夹时，顶部才显示对应文件夹页签，并可关闭该页签。
- 文件夹内图片排序会作为素材导入、文件夹图片节点和对应生成的统一顺序来源。

## 从素材库导入

- 无限画布的图片节点支持从素材库导入整个文件夹。
- 支持在素材导入弹窗中选择整个文件夹，并按素材页中设置的文件夹内顺序显示图片。
- 支持当前文件夹图片全选、取消全选和多选导入。
- 新增“合并显示”开关：开启时，多张图片导入为一个多图图片节点；关闭时，第一张填入当前节点，其余图片分别创建独立图片节点。
- “合并显示”带鼠标悬浮说明，解释合并和拆分导入的差异。
- 多图导入到一个图片节点时，节点内以缩略图网格显示，不显示图片文件名。
- 导入文件夹时，图片节点自动使用文件夹名称作为节点名称。
- 素材导入弹窗中的文件夹区域、标签区域、图片区域支持更灵活的展示和滚动/尺寸调整。

## 无限画布

- 图片节点支持多图集合，用于承载文件夹或多选图片。
- 文件夹图片节点以缩略图网格展示文件夹内图片，并保留素材页中的排序顺序。
- 编排节点的 `@` 命令支持引用上游图片节点、上游文件夹节点中的所有图片，以及文件夹中的单张图片。
- 当上游只有一个文件夹节点时，`@` 菜单简化为“所有图片”和单张图片，不再重复显示文件夹项。
- 支持多选节点后，从其中一个节点右侧连接点发起连接时，其余选中节点一起连接到目标节点。
- 无限画布顶部工具栏新增“导入素材”：默认导入全部已生成图片；选中图片节点时，只导入选中图片节点到素材库。
- 无限画布列表的下载按钮增加“下载所有图”和“仅下载生成图”选项。

## 对应生成与队列

- 编排节点新增“对应生成”模式。
- 当左侧连接一个文件夹节点或多个独立图片节点时，可用同一套提示词按输入图片顺序逐张生成对应结果。
- 对应生成模式下，生成数量由上游图片数量决定，生成数量按钮不可用。
- 对应生成的结果图片按输入图片顺序排布：文件夹节点按从上到下、从左到右的顺序对应生成；多个文件夹节点会分区展示并保留明显间隔。
- 对应生成的单张重新生成，只引用与其对应的那一张参考图，不会引用整个文件夹。
- 大批量对应生成增加前端队列控制：最大并发改为 10，其余任务保持排队状态，前面任务完成后按顺序补位提交。
- 遇到后端排队、频率限制或任务过多提示时，会暂停提交并回退等待，避免一次性把大量任务压入后端导致失败。
- 左下角显示并发数、排队数和队列状态。
- 增加终止生成功能：未选中节点时终止当前画布内全部生成和排队任务；选中生成/排队节点时只终止选中的任务。
- 大批量生成时不再自动选中全部新节点，并优化连接线和画布渲染，减少卡顿。

## 性能与稳定性

- 无限画布连接线渲染改为批量 SVG path，降低大量节点和大量连线时的 DOM 压力。
- 画布状态持久化增加合并写入，减少大批量节点变更时频繁写入本地存储造成的卡顿。
- Agent 聊天流式输出使用动画帧合并刷新，降低高频更新导致的渲染压力。
- 素材库变更会派发同步事件，素材导入弹窗可在排序或文件夹内容变化后刷新显示。

## 部署与运行

- 定制 Dockerfile 和 docker-compose.yml，支持在服务器上通过 Docker Compose 构建、部署和重启应用。
- 部署流程会保留对 `https://cpa.tynote.cn/v1/` 的请求能力；构建后会从容器内检查该地址可达性。
- 项目已配置推送到 `Timeziye/nova-image-studio.git`，后续改动按提交记录持续同步到 GitHub。
